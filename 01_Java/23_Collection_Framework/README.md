### 해쉬 = 분류(속도 빠르게 할려고)  = 군집 = 집합
### 해쉬 알고리즘 = 분류 알고리즘

### 해쉬 알고리즘의 이해
	분류대상 : 
				3, 5, 7, 12, 25, 31
				
	알고리즘을 만들어서			
	분류를 한다. 	
				ex: num % 3
	
	분류를 해 놓으면 탐색의 속도가 매우 빨라진다.	


ex. 해쉬태그
	#기쁜날,  #맛집, #가산디지털	#자바

### hashCode(), equals()를 언제/왜 사용하는지?  ---> Set의 add()를 할때, 동일 여부 판단용
	hashCode() : 속도를 위한 분류 알고리즘 --> 군집을 만든다.
	equals() :  동일 여부 판단

### HashCode()를 호출 하는 곳 ---> Set에서  add  할때 (속도를 위한 분류 알고리즘)
	hashCode() :  속도를 위한 분류 알고리즘 일뿐이고, 동등한 것인지를 판단하지는 않는다.
	equals() :  동일한지를 판단한다.

### hashCode()를 오버라이드 하지 않으면 ?  --->  Set에서 Object것을 호출한다.


### public native int hashCode();
	native의 의미?

#### Object의 hashCode를 사용한다는 의미? : 
	-->  군집을 객체를 만들때마다 1개씩 둔다.
	
#### Override한 hashCode안의  return num % 3 의 의미? : 
	-->  군집을 3개로 만든다.
